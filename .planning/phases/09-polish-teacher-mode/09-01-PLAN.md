## Pedagogical Design

**Teacher-centered features**:
- Overview mode (see all at once)
- Unrestricted navigation (demo any lesson)
- Progress reset (new classes each year)
- Embedded tips (just-in-time help)

**Classroom workflow support**:
1. **Whole-class introduction**: Teacher demos Lesson 1 using Teacher Mode
2. **Individual exploration**: Students work through lessons at own pace
3. **Group activities**: Quizzes solved collaboratively
4. **Hands-on practice**: Mini-apps used for experimentation
5. **Reset for next class**: Teacher uses reset button

**Differentiation support**:
- Advanced students: Jump ahead to Lessons 5-7
- Struggling students: Repeat earlier lessons (no progress penalty)
- ELL students: Visual Kiki support, simple language
- Teachers control pacing (not app-enforced linearity)

## Polish Philosophy

**What "polish" means in this context**:
- Consistency: All buttons behave similarly
- Smoothness: Animations feel natural, not jerky
- Clarity: No confusing UI states
- Delight: Small touches (Kiki reactions, confetti)
- Reliability: No crashes, errors, or broken features

**Areas of focus**:
1. **Visual polish**: Consistent spacing, alignment, colors
2. **Interaction polish**: Hover states, loading indicators, transitions
3. **Content polish**: Typo-free text, grammatically correct German
4. **Performance polish**: Fast load times, smooth animations
5. **UX polish**: Clear navigation, obvious next steps

**Not over-engineered**:
- Simple beats complex (no fancy animations if unnecessary)
- Functional over flashy (educational focus, not gaming)
- Age-appropriate (no overstimulation)
